## Technical Deep-Dive (The 12-Week Curriculum)

<details>
<summary><b>1. Core Infrastructure & Networking</b> (Click to Expand)</summary>
<br>
<ul>
  <li><b>Custom VPC Design:</b> Architecting secure networks with public subnets for web-facing tiers and private subnets for database tiers.</li>
  <li><b>Connectivity:</b> Implementing Internet Gateways (IGW), NAT Gateways for secure outbound traffic, and VPC Endpoints to keep traffic within the AWS network.</li>
  <li><b>High Availability:</b> Configuring Multi-AZ deployments and Elastic Load Balancers to eliminate single points of failure.</li>
  <li><b>Traffic Routing:</b> Managing DNS records and health checks via Route 53 to ensure global application reach.</li>
</ul>
</details>

<details>
<summary><b>2. Security & Compliance</b> (Click to Expand)</summary>
<br>
<ul>
  <li><b>Identity Management:</b> Applying the <b>Principle of Least Privilege</b> using IAM groups and roles, preventing the use of root accounts for daily tasks.</li>
  <li><b>Infrastructure Hardening:</b> Managing stateful Security Groups and stateless Network ACLs to create multiple layers of defense.</li>
  <li><b>Shared Responsibility Model:</b> Understanding the boundary between AWS (Security <i>of</i> the Cloud) and the customer (Security <i>in</i> the Cloud).</li>
  <li><b>Data Protection:</b> Implementing server-side encryption for S3 buckets and utilizing AWS KMS for key management.</li>
</ul>
</details>

<details>
<summary><b>3. Compute, Storage, & Databases</b> (Click to Expand)</summary>
<br>
<ul>
  <li><b>EC2 Lifecycle:</b> Provisioning, configuring, and scaling virtual servers across different instance families.</li>
  <li><b>S3 Mastery:</b> Managing object storage with versioning, static website hosting, and intelligent-tiering for cost savings.</li>
  <li><b>Database Management:</b> Deploying RDS instances with Read Replicas for performance and Multi-AZ for disaster recovery.</li>
  <li><b>EBS & EFS:</b> Selecting appropriate block and file storage based on throughput and IOPS requirements.</li>
</ul>
</details>

<details>
<summary><b>4. Daily Operations & Automation</b> (Click to Expand)</summary>
<br>
<ul>
  <li><b>Linux Administration:</b> Performing daily systems tasks (SSH, grep, sed, chmod, package management) via the command line.</li>
  <li><b>Python Scripting:</b> Utilizing the <b>Boto3 SDK</b> to automate the creation and deletion of AWS resources.</li>
  <li><b>Monitoring:</b> Building CloudWatch Dashboards to visualize infrastructure health and setting SNS notifications for system alerts.</li>
  <li><b>IaC Basics:</b> Introduction to Infrastructure as Code using AWS CloudFormation to ensure repeatable deployments.</li>
</ul>
</details>
