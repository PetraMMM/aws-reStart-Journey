What I’ve Mastered (The 12-Week Journey)
1. Core Infrastructure & Networking
VPC Architecture: Designing and deploying custom Virtual Private Clouds (VPC), including public and private subnets, Route Tables, and Internet Gateways.
Connectivity: Configuring NAT Gateways for secure outbound-only internet access for private instances and using VPC Peering for inter-network communication.
High Availability (HA): Implementing Multi-AZ (Availability Zone) strategies to ensure applications remain online even if a data center fails.
DNS & Traffic Management: Managing domain names and health checks using Amazon Route 53.

2. Security & Compliance
Identity & Access Management (IAM): Mastered the Principle of Least Privilege by creating granular IAM policies, roles, and groups to ensure users only have the access they need.
Shared Responsibility Model: Deep understanding of what AWS secures (the Cloud) vs. what the customer secures (in the Cloud).
Network Security: Implementing "Defense in Depth" using Security Groups (stateful) and Network ACLs (stateless) to protect cloud resources.
Data Protection: Managing encryption at rest and in transit using AWS KMS (Key Management Service) and SSL/TLS certificates.

3. Daily Operations & Automation
Linux Administration: Daily hands-on experience with the CLI, managing file systems, permissions, and package management in Amazon Linux and Ubuntu.
Python for DevOps: Writing scripts to automate routine tasks, such as backing up S3 buckets or monitoring EC2 instance states via Boto3.
Monitoring & Logging: Utilizing Amazon CloudWatch to track metrics and set alarms, ensuring proactive rather than reactive maintenance.
