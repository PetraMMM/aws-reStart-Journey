# Troubleshooting & Managing AWS CloudFormation Deployments

## Overview
In this project, I developed practical skills by building and troubleshooting AWS CloudFormation deployments. 

Because automated deployments rarely go perfectly on the first try in real-world scenarios, this project heavily focused on diagnosing, analyzing, and resolving common deployment errors, infrastructure drift, and deletion failures.

## Objectives Achieved
Throughout this hands-on project, I successfully demonstrated my ability to:
* **Parse JSON Data:** Use JMESPath to query and extract specific values from complex JSON-formatted API responses.
* **Troubleshoot Deployments:** Diagnose failed AWS CloudFormation stack creations using the AWS CLI.
* **Perform Log Analysis:** SSH into a Linux EC2 instance and analyze system log files to determine the exact root cause of a stack creation failure.
* **Detect Infrastructure Drift:** Identify configuration drift when a resource is manually modified outside of the CloudFormation template.
* **Manage Resource Retention:** Troubleshoot a failed stack deletion and apply advanced template configurations (DeletionPolicies) to successfully delete the stack while safely preserving an Amazon S3 bucket and its data.

## Project Breakdown

### 1. JSON Querying with JMESPath
I started by practicing advanced data filtering using JMESPath. This allowed me to easily read and extract specific infrastructure details from the large JSON outputs generated by AWS CLI commands.

### 2. Deployment & Log Analysis
Working from a dedicated `CLI Host` EC2 instance inside a custom VPC, I used the AWS CLI to initiate a CloudFormation stack deployment. The initial deployment intentionally failed. To investigate, I connected to the Linux host and analyzed the internal log files, pinpointing the configuration error preventing the stack from deploying, and applied the necessary fix.

### 3. Drift Detection
To simulate a real-world scenario where an administrator manually changes a setting without updating the code, I manually modified a resource created by the stack. I then ran CloudFormation's **Drift Detection** tool to successfully flag the exact differences between the deployed state and the defined IaC template.

### 4. Deletion Troubleshooting & The S3 Challenge
When tearing down the environment, I encountered a deletion failure because an S3 bucket created by the stack contained objects (AWS prevents deleting non-empty buckets by default). For the challenge, I reconfigured the infrastructure code to alter the stack's deletion policy, allowing me to successfully delete the stack components while safely retaining the S3 bucket and its stored objects.

## Technologies & Tools Used
* **AWS CloudFormation:** Stack deployment, Drift Detection, Deletion Policies
* **AWS CLI:** Command-line infrastructure management
* **Amazon EC2 & VPC:** Networking and compute hosts
* **Amazon S3:** Object storage and retention policies
* **Linux / Bash:** Log file analysis and SSH
* **JMESPath:** JSON query language
